# Cardiac Mastery OS Design Spec

## Overview

Cardiac Mastery OS home should feel like an elite anatomical observatory for an ambitious cardiac trainee. The core subject is an anatomically accurate heart presented in an elevated 3D editorial style, not a textbook diagram and not a photoreal surgical render. The page's single job is to create immediate awe, then resolve into a serious mastery command center.

## Visual Direction

The visual direction is deep red and vascular. The main palette is obsidian black, arterial red, claret, oxidized blood, bone white, and restrained steel gray. The heart is the richest visual object on the page, with controlled glow, depth, and volumetric light. Supporting UI surfaces feel like dark observation panels or surgical instrument readouts rather than standard SaaS cards.

## Home Experience

The home experience is "awe first, then command center." The first screen is a tall immersive hero with the heart slightly off-center, a concise thesis, and one primary next action. As the user moves downward, the heart recedes and the interface resolves from the same space into three main modules: recommended focus, topic mastery, and recent activity. The dashboard should feel derived from the anatomy, not pasted underneath it.

## Motion

Motion is slow, circulatory, and disciplined. The heart has a restrained beat every few seconds, with subtle systolic contraction and faint vascular light response. The initial reveal uses depth, fade, and calibration-like timing. Once the command center is visible, motion becomes quieter and more functional.

## Signature Interaction

The signature interaction is that the heart responds to the trainee's weakest area or current focus. A specific region or vascular path highlights in relation to mastery data, tying the emotional centerpiece directly to the learning system.

## Typography

Typography should pair a controlled editorial display face for the hero with a highly readable sans for the working UI. Small labels should feel precise and instrument-like.

## Non-Goals

- Generic medical blue UI
- Floating dashboard-card SaaS feel
- Overly dramatic horror-like heartbeat
- Decorative anatomy that has no connection to the learning data
